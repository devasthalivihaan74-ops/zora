No manual env var setup needed: since you've already connected Blob storage to this project in the dashboard, Vercel injects `BLOB_READ_WRITE_TOKEN` for you automatically. ## Good to know - **4.5MB limit**: this "server upload" pattern (browser → your function → Blob) is capped at 4.5MB per file, a Vercel platform limit on Serverless Function request bodies. Fine for photos, but if you'll ever need bigger files, Vercel Blob also supports direct browser-to-Blob "client uploads" (up to 5TB) — a different, slightly more involved pattern. Say the word if you want that version instead. - **Public vs private**: `api/upload.js` uses `access: 'public'`, so anyone with the URL can view the file — appropriate for things like product photos. Switch to `access: 'private'` if you need access control. - **No auth on the endpoint**: as written, anyone who can reach `/api/upload` can upload a file to your store. That's fine for an internal/admin tool, but if this button will be public-facing, add an auth check at the top of the handler before calling `put()`.
